get_lat(bw: Bandwidth, unit: str | None = None, freq: Frequency | None = None) → Latency

Latency for provided bandwidth. Linear interpolation is used to calculate latency between two recorded points.

If provided bandwidth is smaller than the smallest recorded sample, the latency corresponding to the smallest recorded bandwidth is returned. The rationale is that curve at this point is usually constant.

If provided banwdith is larger than the largest recorded sample, an exception is thrown. The rationale is that the curve beyond the max bandwidth is exponential and it is difficult to find a good estimate for latency.

Parameters:

  • bw (Bandwidth) – bandwidth

  • unit (str, optional) – units of the returned latency. Defaults to None. If None, latency is returned in the same units as the latencies given at object initialization

  • freq (Frequency, optional) – memory frequency (only needed when latency units are in time units, e.g. ns, us, s)

Returns:

latency for the given bandwidth

Return type:

latency (Latency)
